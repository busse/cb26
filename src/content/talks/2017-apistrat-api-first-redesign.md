Presented at the API Strategy & Practice Conference 2017 in Portland, one of the most influential gatherings for API practitioners. This talk introduced the "APIs for Lemonade Stands" methodology -- a hands-on approach to teaching API-first design using a 1979 Apple II game as a memorable, tangible domain model.

## Key Topics

- **API-first design principles** -- why starting with the API contract changes everything
- **Legacy application assessment** -- identifying where to begin modernization
- **Resource-oriented thinking** -- concern yourself with resources first, then payloads
- **Developer empathy** -- get out of the building and sit with end users

One of two talks delivered at APIStrat 2017.
