Presented at the Austin API Summit, part of the Nordic APIs conference series.

## Key Quote

> "Always follow a top-down approach—start from business need and consumer need, then use that to design APIs."

## Topics Covered

- **API economics** – Understanding the business value of APIs
- **API-first design principles** – Starting with the API contract before implementation
- **Developer empathy** – Designing for the humans who will use your APIs
- **OpenAPI specification** – Using standards to document and communicate API contracts

This talk was a precursor to the keynote I would deliver at the Nordic APIs Platform Summit in Stockholm later that year.
